Startup Instructions for Your Angular App

Prerequisites

  • Node.js: Ensure you have Node.js v18.19.1 or newer installed. Download from Node.js official site.
  • Text Editor: We recommend using Visual Studio Code for Angular development.
  • Terminal: Required for running Angular CLI commands.

Install Angular CLI

To set up Angular CLI globally, open your terminal and run:

npm install -g @angular/cli

Create a New Project

Run the following command in your terminal, replacing ng new <project-name> with the desired name of your Angular project:

ng new <project-name>

OR

ng new <project-name> --routing --style=css

Note: With the first option you will be prompted to configure some project settings (like adding routing or selecting styles) or copy the second code snippet. If unsure, press Enter to accept the default options.

Run Your Angular App Locally

Navigate to your project directory:

cd <project-name>

Start the development server:

ng serve

Once the server is up, visit the app in your browser at:

http://localhost:4200

Setting Up .env in Angular

Using environment variables in Angular can be tricky. I found the following package very helpful for deploying on Vercel with .env:

npm install @ngx-env/builder --save-dev

Once installed, this allows you to easily manage your environment variables for deployment. Make sure to reference your `.env` file correctly in your application.

Manage your sensitive data like API keys securely using the @ngx-env/builder package. Below is a step-by-step guide for setting up environment variables in your Angular app.

1. Install the Package

Run the following command to install the package:

npm install @ngx-env/builder --save-dev

2. Create the .env File

Create a file named .env in the root of your project and add your API key. For example: 

NG_APP_WEATHER_API_KEY=your_api_key_here

Make sure to replace your_api_key_here with your actual Weather API key.

4. Access Variables in Your Code

Use the following syntax to access your Weather API key in a service: 

    import.meta.env.NG_APP_WEATHER_API_KEY

Here’s an example of using the Weather API key in a service: 

private apiKey = import.meta.env.NG_APP_WEATHER_API_KEY;

5. Deployment Setup

When deploying to platforms like Vercel or Netlify, add your environment variables in the platform's settings (e.g., Environment Variables section). Do not commit the .env file to your repository; instead, add it to your .gitignore. 

# .gitignore
.env

This ensures your sensitive data remains secure and is accessible only during deployment.
